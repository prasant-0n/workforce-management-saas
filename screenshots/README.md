# Screenshots Directory

This directory contains screenshots of the WorkForce application pages for the README.md documentation.

## How to Add Screenshots

1. **Start the development server:**
   ```bash
   pnpm dev
   ```

2. **Navigate to each page** and take screenshots:
   - `http://localhost:3000` - Landing page
   - `http://localhost:3000/login` - Login page
   - `http://localhost:3000/register` - Registration page
   - `http://localhost:3000/dashboard` - Dashboard overview
   - `http://localhost:3000/dashboard/leave` - Leave management
   - `http://localhost:3000/dashboard/approvals` - Leave approvals
   - `http://localhost:3000/dashboard/schedule` - Schedule management
   - `http://localhost:3000/dashboard/team` - Team overview
   - `http://localhost:3000/dashboard/users` - User management
   - `http://localhost:3000/dashboard/settings` - Settings

3. **Save screenshots** with the following naming convention:
   - `landing-page.png`
   - `login-page.png`
   - `register-page.png`
   - `dashboard-overview.png`
   - `leave-management.png`
   - `leave-approvals.png`
   - `schedule-management.png`
   - `team-overview.png`
   - `user-management.png`
   - `settings.png`

4. **Recommended settings:**
   - Resolution: 1920x1080 (Full HD)
   - Format: PNG (for better quality)
   - Browser: Chrome/Firefox in light mode
   - Include the full page or main content area

## Tools for Taking Screenshots

- **Browser DevTools**: Right-click → Inspect → Toggle device toolbar for responsive testing
- **Browser Extensions**: Full Page Screen Capture, GoFullPage
- **Native Tools**: macOS: Cmd+Shift+4, Windows: Snip & Sketch, Linux: Flameshot

## Tips for Great Screenshots

- Ensure the page is fully loaded
- Use consistent browser window size
- Capture the main content area (avoid browser UI)
- Show realistic data/content
- Include hover states where relevant
- Test on different screen sizes for responsive design
